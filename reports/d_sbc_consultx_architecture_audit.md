## 5. Phase 5 — why Section 907.2.7 did not surface live

**Confirmed root cause:** the Step 4.2 SEMANTIC_ALIAS for `Mercantile + alarm → 907.2.7` uses a **co-occurrence window of `[\s\S]{0,80}` characters**. Simulation against the failing-case query:

```
query (lowercased):
  index 46:  "محلات تجارية"   ← Mercantile token
  index 120: "رش"              ← sprinkler token   distance 74  → MATCH (within 80)
  index 227: "إنذار"           ← alarm token       distance 181 → NO MATCH (>80)
```

The user's query separates the Mercantile classification (early sentence about "محلات تجارية") from the alarm question (much later sentence "هل يلزم … إنذار حريق؟") by ~180 characters. The 80-char window covers Mercantile + sprinkler but not Mercantile + alarm. The reverse pattern (`إنذار … محلات تجارية`) also fails because alarm appears AFTER Mercantile in this query. PARENT_ALIASES `"907": ["907.2","907.3"]` does not include `"907.2.7"`, so a bare "907" mention also wouldn't pull the new row.

**Failure class:** alias mismatch only. The 907.2.7 row exists in production (`sbc_code_tables`, verified bytes=1841), `fetchStructuredTables` would return it if `extractTableIds` emitted the id, and `buildEvidenceLedger` would surface the `__sbc_table__::SBC-801::907.2.7` chip via Step 4.1. Nothing else is wrong.

**Smallest safe fix (NOT implemented):** widen the window or drop the co-occurrence requirement for the alarm pair only. Two equivalent options:
- **A.** Change both `907.2.7` regexes from `[\s\S]{0,80}` to `[\s\S]{0,400}`. Keeps narrow scope to one paragraph.
- **B.** Replace the two co-occurrence regexes with one boolean rule outside SEMANTIC_ALIASES — i.e. add `907.2.7` to the `tableIds` set whenever the query independently matches a Mercantile pattern AND an alarm pattern (regardless of distance).
- **C.** Add `907.2.7` to `PARENT_ALIASES["907"]` so any mention of section 907 surfaces it, plus a single Mercantile-or-alarm OR rule.

A is the safest and matches the existing co-occurrence shape; B is the cleanest semantically. Option A is one number change in 2 lines.
